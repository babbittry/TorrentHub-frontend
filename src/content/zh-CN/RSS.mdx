## 功能概述

**RSS Feed Token 系统**替代了旧有的全局 `RssKey`，允许用户为不同的 RSS 阅读器（如 qBittorrent, Transmission, Sonarr 等）创建独立的、可精细化管理的订阅 Token。

### 核心优势

- **多 Token 隔离**: 为不同的阅读器或设备创建独立的 Token，泄露后可单独撤销，不影响其他订阅。
- **细粒度控制**: 每个 Token 均可配置订阅类型、分类、结果数量和过期时间。
- **使用监控**: 系统会记录每个 Token 的使用次数、最后使用时间、IP 地址和 User-Agent，方便追踪。
- **安全设计**: Token 泄露不影响主账号安全。

## 如何使用

### 1. 创建 RSS Token

1.  访问 **用户中心 > RSS 订阅管理**。
2.  点击“创建新的 Token”按钮。
3.  填写表单：
    - **名称**: 为你的 Token 起一个容易识别的名字（例如：“我的 qBittorrent”）。
    - **订阅类型**: 选择你需要的 Feed 类型（最新、分类、收藏等）。
    - **分类过滤**: （可选）选择只订阅特定的种子分类（如电影、电视剧）。
    - **最大结果数**: （可选）设置 RSS Feed 返回的条目数量（默认 50）。
    - **过期时间**: （可选）为 Token 设置一个自动失效的日期。
4.  点击“创建”。

### 2. 获取 RSS URL

创建成功后，系统会生成一个包含新 Token 的 RSS URL。

- **格式**: `https://your-site.com/api/rssfeed/feed/{token}`
- 点击“复制”按钮，将此 URL 粘贴到你的 RSS 阅读器中。

**重要**: 此 URL 无需登录即可访问，请妥善保管，不要泄露。

### 3. 管理你的 Tokens

在 **RSS 订阅管理** 页面，你可以：

- 查看所有已创建的 Token 列表。
- 监控每个 Token 的使用情况（使用次数、最后使用时间等）。
- **撤销** 任何你怀疑已泄露或不再使用的 Token。
- **撤销所有** Token 以快速禁用所有 RSS 订阅。

## 常见问题 (FAQ)

### Q: 我应该为每个设备都创建一个新的 Token 吗？

**A:** 是的，强烈建议这样做。为你的家庭服务器、笔记本电脑、手机等分别创建独立的 Token。这样，如果其中一个设备的配置泄露，你只需撤销对应的 Token，而不会影响其他设备。

### Q: `categoryFilter` 字段是如何工作的？

**A:** 你可以选择一个或多个种子分类。设置后，你的 RSS Feed 将只包含这些分类下的最新种子。

### Q: 如果我的 Token 泄露了怎么办？

**A:** 立即登录网站，进入 **RSS 订阅管理** 页面，找到泄露的 Token 并点击“撤销”按钮。然后，为你的设备创建一个新的 Token 并更新你的 RSS 阅读器配置。

### Q: 撤销 Token 会影响我的账号吗？

**A:** 不会。撤销 Token 只会使对应的 RSS URL 失效，不会对你的账号、分享率或已下载的种子产生任何影响。
